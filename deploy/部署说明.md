# 音乐工坊 · 部署到 music.yongle.school

> 目标：把产品部署到 `yongle.school` 的子域名（推荐 `music.yongle.school`）。
> 关键：本产品**不是纯静态**——`/api/plan`、`/api/compose` 需要 Node 后端（持有 MIMO 密钥）。所以要在服务器跑 `serve.mjs`，用 nginx 反代 + 子域名。

服务器（DNS 现状）：`yongle.school → 43.159.171.3`（腾讯云）。**子域名继承 yongle.school 的 ICP 备案**，所以 `music.yongle.school` 走国内不会被 ICP 墙拦（前提：yongle.school 已备案且绑这台机器）。

---

## 前置：两件只有你能做的事
1. **GitHub 推送**需要你先认证：`gh auth login`（浏览器一次）。认证后我可以 `gh repo create music-workshop --public --source=. --remote=origin --push`，或你自己跑这行。
2. **服务器访问**：部署要 SSH 进 `43.159.171.3`。你可以自己按下面跑，或授权我访问（给我确认的主机 + 免密方式）后我来跑。

## 一次性部署（在服务器上）

```bash
# 0) DNS：给 music 加一条 A 记录 → 43.159.171.3（在你的 DNS 面板）

# 1) 取代码
sudo mkdir -p /opt/music-workshop && sudo chown -R $USER /opt/music-workshop
git clone https://github.com/<你的账号>/music-workshop.git /opt/music-workshop
cd /opt/music-workshop

# 2) 放密钥（.env 不在仓库里，手动创建，chmod 600）
cp backend/.env.example backend/.env
#   编辑 backend/.env，填入 MIMO_API_KEY（就是本地那把 tp-... 的）
chmod 600 backend/.env

# 3) 构建 dist（音频外置 + 懒加载版 HTML）
node backend/extract-samples.mjs CubCopCat.src.html backend/dist

# 4) 起后端为常驻服务
sudo cp deploy/music-workshop.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now music-workshop
curl -s -o /dev/null -w "本地 8080 → %{http_code}\n" http://127.0.0.1:8080/   # 期望 200

# 5) nginx 反代 + 子域名
sudo cp deploy/nginx-music.conf /etc/nginx/sites-available/music.yongle.school
sudo ln -sf /etc/nginx/sites-available/music.yongle.school /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# 6) HTTPS 证书（子域名已备案 → 80 口能签）
sudo certbot --nginx -d music.yongle.school
```

完成后访问 **https://music.yongle.school**。

## 安全 & 注意
- **MIMO 密钥只在服务器 `backend/.env`（600）**，绝不进仓库、不进客户端。`serve.mjs` 才用它。
- 后端只监听 `127.0.0.1:8080`，唯一公网入口是 nginx。
- `/api/compose` 长曲要几十秒，nginx 已设 `proxy_read_timeout 180s`。
- 若 `yongle.school` 其实**没在这台机器备案**：国内会被 ICP 墙拦，需换一个"已备案且绑这台机器"的域名的子域名（参考深脑用 `sb.ai.zaowuyun.com` 的做法）。
- 首屏 1.4MB HTML + 懒加载音频；`samples-core.js` 与 `assets/` 由第 3 步生成（不在仓库）。
