# 音乐工坊 · K12 虚拟音乐工坊：知识结构与成长主线设计

> 目标：把当前"功能很多但平铺、无进阶、无分龄"的产品，整合成一条 K12 可用的**创作者成长主线**。
> 核心理念（创造者教育）：孩子是**作者**，一路走到"署名作品"。

---

## 一、现状诊断

现在是 **5 个并行筒仓，没有脊椎**：

| 域（对标艺术课标） | 现有功能 | 强弱 |
|---|---|---|
| 🎧 听（审美感知） | Quiz 一部分 | 弱 |
| 🎹 奏（艺术表现） | 150 乐器 + 写实控制器 | 强 |
| ✍️ 创（创意实践） | Medly + AI 共创 | 强 |
| 📖 识（文化理解） | 图书馆 + 乐理小课 + 图谱 | 散 |

问题：**无学习路径、无分龄、无评价闭环**。K12 跨 6–18 岁，能力差巨大，却共用一个平铺入口。图谱的碰撞，本质也是"把 106 个知识点平铺"的同一病症。

---

## 二、成长主线（脊椎）★最重要

把散落的功能串成一条**看得见进度的路径**，用现有 `Tasks/Quest/徽章`承载：

```
① 认识声音   听音色、辨高低快慢          ← 扩 Quiz + 听力训练（补"听"）
② 弹一个乐器  用写实控制器弹出旋律        ← 已有：钢琴/吉他/提琴/鼓
③ 懂一点乐理  音阶/节奏/和弦             ← 已有：乐理小课 + 音阶高亮
④ 创作一小段  AI 共创起草 → 自己改        ← 已有：AI 共创 + Medly
⑤ 完成署名作品 存进作曲台、署上名字        ← 接后端：作品云存储
⑥ 发表 / 合奏  分享链接 / 班级作品展       ← 接后端：分享 + 教师侧
```

每完成一级点亮一颗星；**每级产出一件署名作品**。这条线就是"从演奏者到作者"的创造者教育落地。

---

## 三、分龄分级（L1–L3）

同一套乐器，用"难度开关"覆盖全 K12：

| 级 | 对象 | 表现 |
|---|---|---|
| **L1 启蒙**（低幼/小学低） | 真实控制器 + **音名提示常显**、少而精的乐器、以"玩/听"为主 | 你刚做的音名按钮正好用上 |
| **L2 进阶**（小学高/初中） | 隐藏部分提示、开放乐理与 Medly、引导创作一段完整旋律 | |
| **L3 创作**（初中/高中） | 全乐理、多轨编曲、合奏/发表、作品集 | |

首页据此从"150 张卡的墙"变成 **「继续你的旅程」（路径）+「自由探索」（画廊）** 两个入口。

---

## 四、图谱 → 学习地图（同时根治碰撞）

当前碰撞的根因：106 个乐器平铺，物理上塞不下。**改成分层**：

- 顶层：**四域 或 乐器族群**（8 个大节点）——空间充裕、零碰撞。
- 点开一个族群 → 展开其中乐器（十几个，能排开）。
- 叠加**进度**（已开锁/已掌握），让它从"关系图"变成"**课程地图**"。

这既解决碰撞，又让图谱有了教学意义（你在地图上看到自己学到哪）。

---

## 五、对标课标 + 评价成果（B2B2C 抓手）

- **对标**《义务教育艺术课程标准(2022)》四大素养，让学校/教培能当**正式课**采用。
- **评价**：每级一件署名作品 + 教师看进度与作品（接后端 Phase 2）。
- 这两点把产品从"遊具"变成"可交付、可采购的课程"。

---

## 六、落地顺序（建议）

1. **成长主线 MVP**：用现有 Tasks 搭出 ①→⑤ 的一条线（不新增大功能，只做串联 + 进度）。
2. **分级开关 L1/L2**：提示显隐 + 乐器/乐理的解锁。
3. **图谱分层**：族群→展开 + 进度叠加（顺带彻底修碰撞）。
4. **接后端**：署名作品云存 + 分享 + 教师侧 → 打通⑤⑥，跑一个真实班级。

> 先做 1（脊椎），是让"已经很强的一堆功能"第一次变成"一门课"的关键一步。
