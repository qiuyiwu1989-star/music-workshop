# CubCopCat 轻后端设计（独立产品 · PocketBase）

> 目标：把「作品锁在本机」的单文件应用，升级为**机构-班级-学生三级账号 + 作品云端存储 + 教师后台**的可交付教育产品。
> 原则：**独立部署，不碰中台 `zhongtai_app` 库**；保持前端单文件离线可用；未成年人数据最小化。

---

## 0. 选型结论

**PocketBase** = 一个 Go 单文件二进制 + 一个 `pb_data/` 目录（内嵌 SQLite + 文件存储）。

| 需求 | PocketBase 能力 |
|---|---|
| 三级账号鉴权 | 内置 auth collection + `role` 字段 |
| 作品存储 | `data`(json，存 Medly 工程) + `audio`(file，存导出音频) |
| 权限隔离（学生看自己 / 老师看本班） | Collection API Rules（等价 RLS） |
| 老师后台 | 自带 Admin UI（Phase 1 可先顶用） |
| 前端接入 | 官方 JS SDK，浏览器直连 |
| 运维 | 一个二进制 + systemd；备份 = 复制 `pb_data/` |

规模判断：SQLite 对「几十~几千学生、偶发写入」绰绰有余。真到几万并发再迁 Postgres，**数据结构不用变**。

## 1. 数据模型（collections）

### `users`（内置 auth 类型，扩展字段）
| 字段 | 类型 | 说明 |
|---|---|---|
| username | text（内置） | 登录名。**用户名登录，不强制邮箱**（未成年人合规） |
| password | 内置 | |
| name | text | 显示名 |
| role | select(`student`,`teacher`,`org_admin`) | 默认 `student` |
| org | relation → orgs | 所属机构（可空，Phase 2 用） |
| klass | relation → classes | 所属班级（学生用，可空） |

> 配置要点：把 `email` 设为**非必填**，鉴权 identity 用 `username`。学生账号由老师批量生成，不收手机/身份证/邮箱。

### `orgs`（机构）
| name(text，必填) | note(text) |

### `classes`（班级）
| name(text) | org(relation→orgs) | teacher(relation→users) | term(text，如 "2026春") |

### `works`（作品）★核心
| 字段 | 类型 | 说明 |
|---|---|---|
| student | relation → users | 作者 |
| klass | relation → classes | 所属班级（可空） |
| title | text | 作品名 |
| data | json | **Medly 工程**：`{name,bpm,bars,beatsPerBar,snap,tracks:[...]}` |
| audio | file（单个，可空） | 导出的音频/视频，Phase 3 用 |
| status | select(`draft`,`submitted`,`published`) | 默认 `draft` |

### `progress`（学习进度，迁移现有 Tasks/Quest，Phase 2+）
| student(relation→users) | lesson_key(text) | state(json) |

## 2. 权限规则（API Rules）★命门

在 PocketBase 里，规则是写在 collection 上的表达式。**`works` 的规则最关键**：

| 操作 | Rule 表达式 |
|---|---|
| List / View | `@request.auth.id = student.id \|\| klass.teacher = @request.auth.id \|\| @request.auth.role = "org_admin"` |
| Create | `@request.auth.id != "" && student = @request.auth.id` |
| Update | `student = @request.auth.id` |
| Delete | `student = @request.auth.id` |

- 学生：只能增删改查自己的作品。
- 老师：能 **List/View 本班**所有学生作品（靠 `klass.teacher`）。
- 机构管理员：能看本机构（`role=org_admin`）。

`users` collection：
- Create Rule：Phase 1 允许自助注册 → `""`（公开）；Phase 2 收紧为「仅老师/admin 建号」。
- Update Rule：`id = @request.auth.id`（只能改自己）。

`classes` / `orgs`：List/View 给 `@request.auth.id != ""`；写操作仅 `role` in (`teacher`,`org_admin`)。

## 3. 作品存储策略

- **Medly 工程 = 小 JSON** → 直接进 `works.data`（jsonb）。当前工程结构（读自现有代码）：
  ```json
  { "name": "...", "bpm": 120, "bars": 8, "beatsPerBar": 4, "snap": 0.25,
    "tracks": [ { "name":"", "instrumentId":"piano", "color":"", "muted":false, "solo":false,
                  "notes":[ /* {...} */ ] } ] }
  ```
- **导出音频/视频 = 大文件** → `works.audio`（PocketBase 自带文件存储，无需额外 S3）。Phase 3 再做。

---

## 4. 离线优先（教室网络不稳，必做）

1. **演奏 / 创作全程离线**：现有能力一行不改。
2. 作品先写 `localStorage`（现有 `vi_medly_projects` 保留为**草稿层**）。
3. 联网且登录时，**按 `updated` 时间戳同步**到 PocketBase（last-write-wins）。
4. 每条云端作品在本地缓存一份 `cloudId`，用于覆盖更新而非重复创建。

## 5. 前端接入点（映射到现有 HTML 代码）

前端是单文件，**不直接改巨型文件**，而是：把 `cubcopcat-backend.js` 内联进去 + 改 4 个接入点。

| 接入点 | 现状 | 改法 |
|---|---|---|
| 引入 SDK | 无 | 在 `</body>` 前内联 `pocketbase.umd.js`（~50KB），再内联 `cubcopcat-backend.js` |
| `Auth` 对象（约 4759 行） | localStorage 玩具账号 | 载入后由 `CloudAuth.attach()` 覆盖 `register/login/logout`，接口签名不变 |
| `Medly.saveProject()`（约 15926 行） | 只写 localStorage | 保留本地写；成功后追加 `Backend.syncWork(project)` |
| Medly 工具条（约 4076 行 `💾 Save` 附近） | 只有本地保存/加载 | 加两个按钮：`☁️ 云端保存` → `Medly.saveToCloud()`；`☁️ 我的作品` → `Medly.showCloudDialog()` |

> `CloudAuth.attach()` 保持 `Auth.register(name,pw,confirm)`、`Auth.login(name,pw)`、`Auth.logout()`、`Auth.isLoggedIn()`、`Auth.displayName()`、`Auth._afterEnter()`、`Auth.continueAsGuest()` 全部签名不变——所以 `submitAuth`、`updateAccountUI` 等**调用方零改动**。

详细集成代码见同目录 `cubcopcat-backend.js` 顶部注释。

## 6. 部署（独立 · 国内）

```bash
# 1. 下载 PocketBase 二进制（对应你的服务器架构）放到独立目录
mkdir -p /opt/cubcopcat && cd /opt/cubcopcat
# 放入 pocketbase 可执行文件

# 2. 首次启动（本地回环，避免裸暴公网）
./pocketbase serve --http=127.0.0.1:8090

# 3. 建 superuser（首启会提示，或）
./pocketbase superuser create <你的管理员邮箱> <强密码>

# 4. 跑 setup 脚本建 collections + 规则（见 setup-pb.mjs）
node setup-pb.mjs
```

- **systemd 常驻** + **nginx 反代** + **certbot 证书**（模板见第 7 节）。
- **ICP 墙（国内服务器躲不掉）**：用**已备案、且备案绑这台机器**的域名的子域名指向，如 `api.<已备案域名>`。DNS A 记录 → 服务器 IP；nginx `server_name` 用它；certbot 对它签证书（已备案 → 80 口不被拦，能签）。
  - 验证（**从国内网络测**）：`curl -sS -o /dev/null -w "%{http_code}\n" https://api.<已备案域名>/api/health` → 200 才算通。
- PocketBase **只监听 `127.0.0.1:8090`**，唯一公网入口是 nginx。

### 未成年人数据合规
- 数据全在自己 `pb_data/`（数据主权）。
- 学生账号**老师批量生成**：用户名+密码，**不收手机/邮箱/身份**。
- 作品默认 `draft` 私有；`published` 需显式动作（老师/家长同意）。
- 定时备份：`cp -r pb_data pb_data.bak.$(date +%F)` 或 rsync 到异地。

## 7. nginx 反代模板

```nginx
server {
  listen 443 ssl http2;
  server_name api.你的已备案域名;

  ssl_certificate     /etc/letsencrypt/live/api.你的已备案域名/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/api.你的已备案域名/privkey.pem;

  client_max_body_size 50m;          # 允许作品音频上传

  location / {
    proxy_pass http://127.0.0.1:8090;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;      # PocketBase realtime(SSE/WS)
    proxy_set_header Connection "";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 300s;
  }
}
```

## 8. 落地顺序

| Phase | 内容 | 产出 |
|---|---|---|
| **0** | 起 PocketBase + `setup-pb.mjs` 建 collections/规则 | 后端可用（含 Admin 后台） |
| **1** | 内联 SDK + `CloudAuth.attach()` + Medly 云存储 | **作品跨设备，不再锁本机** |
| **2** | orgs/classes + 批量建学生账号脚本 + 老师看本班 | **能上课、看成果** |
| **3** | 作品发布/分享页 + 音频导出上传 | **招生抓手** |

**先冲到 Phase 1**——它解决最痛的「作品锁本机」。
