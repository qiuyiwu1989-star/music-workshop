# AI 共创 · 推理逻辑说明书

> 版本对应提交 `9931648` 一线。描述"孩子说一句话 → 得到一整首可演奏的曲子"背后的完整推理链路。

---

## 0. 设计理念（为什么这样推理）

- **创造者教育**：AI 是**共创助教**，不是代笔。孩子是**作者**——AI 只出"草稿 + 解释"，孩子再改、署名。
- **像 Deep Research 一样"先想清楚再动手"**：不把一句话直接丢给模型出音符，而是先**深度理解意图 + 定作曲构思**（并展示给孩子看），再**照这份构思谱写整首**。惊喜感来自"AI 真的懂我、还想好了怎么写"。

---

## 1. 总体推理流程（两段式）

```
孩子输入一句意图 + 选一个乐器(音色)
        │
        ▼
┌─────────────────────────────┐  阶段①：理解 + 构思
│  POST /api/plan             │  （像 Deep Research 的"先规划"）
│  → 深度理解 + 作曲构思 JSON   │  展示：💡我懂了 / 🎯构思 / 🎛️手法
└─────────────────────────────┘
        │  把构思(调式/速度/结构/手法)带下去
        ▼
┌─────────────────────────────┐  阶段②：照构思谱写
│  POST /api/compose          │  严格用构思的 调式/速度/结构/手法
│  → 一整首长旋律的音符 JSON    │  120-200+ 个音
└─────────────────────────────┘
        │
        ▼
   前端播放（按 BPM 定时触发）+ 键盘/指板同步点亮 + 可停止
        │
        ▼
   "存进作曲台" → Medly 新建一条 track，孩子接手编辑、署名
```

两段都由 **MIMO（mimo-v2.5，Anthropic 兼容）** 完成；密钥只在服务器侧，客户端从不接触。

---

## 2. 阶段①：深度理解 + 作曲构思（`/api/plan`）

**目的**：读懂孩子"没说出口的那层"，并把它翻译成一份可执行的作曲构思。

- **输入**：`{ intent, instrumentId, instrumentName }`
- **模型调用**：MIMO，`thinking:disabled`（直接出干净 JSON，快≈7-8s），`max_tokens: 1400`
- **系统角色（PLAN_SYSTEM）**：策划师——先深度理解，再定构思。
- **输出（服务器已做长度/范围清洗）**：
  ```json
  {
    "understanding": "你其实想表达…（温暖、具体，戳中那种感觉）",
    "mood": "情绪词",
    "genre": "风格",
    "scale": "如 C_major / A_minor",
    "bpm": 40-160,
    "structure": "如 引子-主歌-副歌-尾声",
    "devices": ["上行琶音", "切分节奏", "长音铺垫", …≤6],
    "plan": "我打算这样写…（给孩子看的一句人话）"
  }
  ```
- **前端展示**：`💡 我懂了：{understanding}` + `🎯 {genre}·{scale}·{bpm}BPM·{structure}` + `🎛️ 手法：{devices}` + `{plan}`。

> 这一步就是"惊喜感"的来源：孩子先看到 AI 的理解与构思，再听结果。

---

## 3. 阶段②：照构思谱写整首（`/api/compose`）

**目的**：把构思落成一整首可演奏的、较完整的长旋律。

- **输入**：`{ intent, instrumentId, instrumentName, plan, history }`
- **模型调用**：MIMO，`thinking:disabled`，`max_tokens: 12000`（容纳 ~300 音的长 JSON）
- **系统角色（COMPOSE_SYSTEM）关键约束**：
  - 长度**硬性**：120-200 个音、约 24-48 小节（"一首完整的歌，不是片段"）。
  - 结构：有段落感（A-B-A / 主歌-副歌）、动机重复/模进/变化发展、结尾收束。
  - 节奏多样（0.25/0.5/1/2 拍混用）。
  - **单声部**：音符不重叠、按 start 递增；所有 pitch 落在所选 scale 内。
  - 适配乐器音区与特点。
- **把构思带进 prompt**：附加一句"已定好的构思（请严格遵循）：调式 X、速度 Y、风格、结构、手法…用这个调式与速度写成整首"。→ **保证 plan 与 compose 一致**（实测：plan 定 E-flat/72，compose 也是 E-flat/72）。
- **输出**：
  ```json
  { "bpm":40-160, "scale":"…", "notes":[{pitch,start,duration,velocity}…], "why":"一句给孩子的话" }
  ```
- **健壮性两道闸**：
  1. `parseComposition`：长输出若被截断导致 JSON 不完整，**截到最后一个完整音符**再补齐 `]}` 挽救。
  2. `clampNotes`：逐音符夹取到合法范围（pitch 21-108、duration≥0.1、velocity 1-100），并按 start 排序。

---

## 4. 数据契约

**音符 Note**（贯穿谱写、播放、Medly）：
| 字段 | 含义 | 范围 |
|---|---|---|
| pitch | MIDI 音高（60=中央 C4） | 21–108 |
| start | 起始拍（浮点） | ≥0 |
| duration | 时值（拍） | ≥0.1 |
| velocity | 力度 | 1–100 |

**曲子** = `{ bpm, scale, notes[], why }`；**构思** = 见 §2。

---

## 5. 播放与交互逻辑（前端）

- **音色**：面板下拉，默认**跟随当前打开的乐器**，可切任意旋律乐器；生成的曲子用选中乐器的真实音色演奏。
- **播放（playMelody）**：按 `每拍秒数 = 60/bpm` 把每个音符排成两个定时器——起音 `AudioEngine[乐器.play](pitch)`＋点亮，抬起 `AudioEngine.release(pitch)`＋熄灭（鼓类只点亮 120ms）。
- **同步动效**：按当前控制器点亮——钢琴键 / 吉他弦(瞬闪) / 提琴指板格。
- **停止**：播放键切换为"⏹ 停止"，点击立即清掉所有定时器 + **释放正在发声的音**（长音不再停不下来）+ 熄灭；播完自动复位为"▶ 再听一次"。
- **存进作曲台**：把 notes 原样建成 Medly 一条 track（作者接手编辑、署名）。
- **多轮**：`history` 保存对话（意图 + 构思 + why），供后续"再深一点/换个感觉"带上下文。

---

## 6. 模型与基建

- **模型**：MIMO `mimo-v2.5`，Anthropic 兼容端点（`/v1/messages`，`x-api-key`）。
- **两处都关 thinking**：因为要稳定产出 JSON；"深度思考"通过 plan 的 understanding/结构字段**显性表达**，而非隐藏的思维链。
- **安全**：密钥仅在服务器 `serve.mjs` 读 `.env`；客户端 → 后端 `/api/plan`·`/api/compose` → MIMO。密钥绝不进客户端 HTML。
- **离线优先**：演奏/播放本地进行；只有 AI 生成需要联网。

---

## 7. 已知局限 & 扩展点（下一步可加）

| 现状 | 可扩展 |
|---|---|
| 只"深度思考"，**不联网** | 阶段①前加"联网查证风格/参考曲"，再构思（需搜索 API） |
| 信息不足时**直接猜** | 阶段①允许**反问一句**（缺关键信息先问再写） |
| **单声部**旋律 | 多轨/和声/伴奏（planner 规划多个声部，分别 compose） |
| 无自评 | 加"自评→打磨"闭环（compose 后让模型对照 plan 检查并修订） |
| 停止=回到开头 | 真"暂停/继续"（记住已播拍、重排剩余音） |
| plan/compose 各调一次 MIMO | 需要更强时用"多候选 + 择优"（生成 3 版，孩子/模型选一版） |

---

## 8. 一句话总结

**理解 → 构思（展示）→ 照构思谱写整首（护栏兜底）→ 用所选乐器演奏（可停）→ 存成孩子的作品。**
核心是"先想清楚再动手 + 全程让孩子当作者"。
